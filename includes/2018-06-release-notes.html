Compatibility notes

This version of the Sharemind SDK is not compatible with older versions of the Sharemind SDK, including on the network protocols, configuration and API/ABI levels.

Features

Emulator

  • Sharemind Emulator now provides a separate Python program called argument-stream-decipher.py that can be used to decipher the output stream of SecreC program results. Sharemind Emulator output should be passed as standard input for the argument-stream-decipher.py.

SecreC compiler and standard library

  • SecreC now allows numeric literals to be annotated by type, e.g. instead of 0 :: uint64 it's now also possible to directly write 0u64. Supported numeric literal suffixes are:
    • | Suffix | Type    |
+--------+---------+
| u8     | uint8   |
| u16    | uint16  |
| u32    | uint32  |
| u64    | uint64  |
| i8     | int8    |
| i16    | int16   |
| i32    | int32   |
| i64    | int64   |
| f32    | float32 |
| f64    | float64 |
  • Improved leakage information in the documentation of several SecreC standard library functions.
  • Significantly improved the performance of SecreC assembler when compiling large programs. As a trade-off, memory usage is increased.

Fixed issues

  • Fixed an issue where SecreC programs compiled in heterogeneous environments could not be invoked as they had differing hash values. As a result of this fix, SecreC bytecode runtime error messages now only contain basename of the relevant SecreC source file and not the full path. To go back to the old behaviour for more convenient debugging, use the --runtime-error-path-style=fullpath option in the SecreC compiler command-line.
  • Sharemind Emulator module mod_shared3p_emu now correctly fails with "invalid configuration" error instead of the "general error" when its configuration file is missing.
  • Fixed an issue where Sharemind Emulator exited with segmentation fault when running SecreC programs that contain keydb or tabledb syscalls as a user that is not defined in the User section of the access policy.
  • Fixed incorrect and crashing behavior of Sharemind Emulator on passing process arguments from command-line.
  • Fixed Sharemind Emulator incorrectly parsing empty long options (e.g. --str=).
  • If Sharemind Emulator fails to open a file for logging, it now also displays the file name in the error message.
